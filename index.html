1. Introduction & Overview

CyberLynX is a comprehensive security tool designed to orchestrate, automate, and centralize security scanning and vulnerability analysis. It provides a unified interface for security professionals to manage distributed tools and analyze security data efficiently.

🛠 Integrated Security Stack

Network Discovery

Nmap, Naabu

Vulnerability Scanning

Nuclei, Nikto, Trivy

Reconnaissance

Subfinder, HTTpx

Fuzzing & Crawling

FFuf, Katana

Analysis

CVEMap, Wappalyzer

Exploitation

Hydra, SQLMap

🚀 Key Features

  • Multi-Architecture Support Native builds for both linux/amd64 and linux/arm64.
  • Enterprise CI/CD Fully automated pipeline with Jenkins integration.
  • Docker Optimized Multi-stage, performance-tuned Docker builds.
  • Scalable Architecture Go-based backend with a high-performance React frontend.

2. Getting Started

Prerequisites

Runtime Go 1.25+
Environment Node.js 20+
Containerization Docker (Buildx)

🐳 Quick Start with Docker

# Build the multi-arch image
docker build -t cyber-lynx:latest .
# Run the local environment (Dual-container mode)
./start-containers.sh
Frontend http://localhost:3000
Backend API http://localhost:8080

💻 Local Development

Backend

cd backend
go mod download
go run main.go

Frontend

cd frontend
npm install
npm start

3. CI/CD Pipeline

CyberLynX includes a robust **Jenkinsfile** that automates the entire lifecycle of the application.

Diagnostics

Environment and tool verification across multi-arch nodes.

Backend CI

Dependency management, security linting, and build validation.

Frontend CI

Automated production-optimized builds and artifact preservation.

Docker Orchestration

Multi-arch builds pushed directly to Docker Hub.

CyberLynX is designed with a **Privacy-First** architecture:
