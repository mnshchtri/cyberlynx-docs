CyberLynX: Official Documentation

Welcome to the official documentation for CyberLynX, a self-hosted, centralized penetration testing orchestration platform designed for independent security researchers, freelance testers, and academic enthusiasts.

Get Started Deployment Guide

1. Introduction & Overview

What is CyberLynX?

CyberLynX is an advanced web application engineered to centralize and automate the penetration testing workflow. It directly addresses the inefficiencies of "tool-hopping" and manual data aggregation by providing a unified environment where practitioners can manage target infrastructures, orchestrate disparate command-line tools, and fully automate the reporting process.

Unified Dashboard

Manage engagements, projects, and target scopes (IPs, domains, URLs) from a single interface.

Tool Orchestration

Execute industry-standard CLI tools (Nmap, Nikto, SQLMap, etc.) directly from the web UI.

Workflow Chaining

Pass tool outputs directly into next-step inputs (e.g., Subfinder output piped to Nmap).

Automated Reporting

Aggregate normalized JSON findings to automatically generate professional HTML and PDF reports.

Tech Stack

Frontend React.js SPA
Backend Go / Golang
Database MongoDB Atlas

2. Getting Started (Deployment Guide)

Prerequisites

Ensure your host environment meets the following requirements:

  • Go (v1.20 or higher)
  • Node.js & npm (v16+)
  • MongoDB Atlas URI
  • Tools in system $PATH

Installation Steps

1. Clone the Repository

git clone https://github.com/mnshchtri/cyber-lynx-go.git
cd cyber-lynx-go

2. Backend Setup (Go)

Create a .env file in the backend root:

PORT=8080
MONGO_URI=mongodb+srv://<username>:<password>@cluster0.mongodb.net/cyberlynx
JWT_SECRET=your_super_secret_jwt_key
cd backend
go mod tidy
go run main.go

3. Frontend Setup (React)

cd ../frontend
npm install
npm start

The React SPA will open at http://localhost:3000.

3. User Guide

Authentication & Profiles

CyberLynX utilizes stateless JSON Web Tokens (JWT) and bcrypt password hashing for secure access. Upon login, the JWT is stored securely in memory to mitigate XSS vulnerabilities.

"Use the Profile tab to update your professional metadata which will be appended to your generated reports."

Project & Target Management

All engagements in CyberLynX are isolated into Projects. Define your scope by adding Targets (IP addresses, CIDR ranges, subdomains, or URLs).

Executing Scans & Workflow Orchestration

To build a workflow, select multiple tools in sequence. For example, configure Subfinder to run first, and configure Nmap to ingest the JSON output of Subfinder's discovered domains.

> Workflow Engine Status

Automated Reporting

The engine consolidates findings, categorizes them by severity (Critical, High, Medium, Low), and exports fully formatted HTML or PDF documents.

4. Developer Guide (Extensibility)

If you have custom Bash scripts or niche tools, you can integrate them using our **Modular Adapter Pattern**.

Writing a Custom Go Adapter

adapters/interface.go Golang 
type ToolAdapter interface {
    Execute(target string, params []string) (RawOutput, error)
    Parse(output RawOutput) ([]NormalizedFinding, error)
}

Execute Method

Write the logic to securely trigger the CLI binary using Go's os/exec package. Whitelist parameters to prevent injection.

Parse Method

Map raw CLI output (text/XML) to internal NormalizedFinding JSON structs for MongoDB storage.

5. Security & Best Practices

⚠️ WARNING: CyberLynX executes offensive security tools. Ensure explicit authorization before scanning targets.

Sandboxing

Run the Go backend inside a restricted Docker container to limit the blast radius of any potential tool exploitation.

Database Security

Restrict MongoDB Atlas cluster IP access list only to your static hosting environment. Never use 0.0.0.0/0.

HTTPS/WSS

Always deploy behind a reverse proxy (Nginx/Traefik) with TLS/SSL for end-to-end encryption.
